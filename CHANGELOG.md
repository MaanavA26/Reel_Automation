# Changelog

All notable changes to Reel Automation are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Entries under `[Unreleased]` describe changes that have landed on `main` but
are not yet part of a tagged release. When cutting a release, move the
`[Unreleased]` section under a new version heading with the release date.

## [Unreleased]

### Added

- Apache License 2.0. `LICENSE` file at repo root; SPDX metadata in `backend/pyproject.toml` and `frontend/package.json`; reference in README.
- Deep Research state and provenance schema (`backend/app/schemas/research_state.py`) — `ResearchState`, `KnowledgeAcquisitionState`, `Source`, `Chunk`, `Evidence` with attached (inline) provenance, type-prefixed opaque IDs, and timezone-aware UTC timestamps. Decisions documented in [ADR 0001](docs/adrs/0001-research-state-and-provenance.md). First Phase 0 feature.
- Research Control band schema (`ResearchPlan`, `SubQuestion`) added to `backend/app/schemas/research_state.py`; `ResearchState` now includes a `plan` substate alongside `acquisition`. Second Phase 0 feature.
- Deep Research workflow skeleton (`backend/app/workflows/deep_research.py`) — a compiled LangGraph graph wiring `ResearchState` through async lifecycle-stub nodes (plan → acquire → reason → publish) that runs end-to-end (`queued → completed`). Establishes the node I/O contract (partial-state-update returns) every later agent/tool node plugs into. Decisions and the empirically-grounded fan-out deferral documented in [ADR 0002](docs/adrs/0002-langgraph-workflow-integration.md). Adds the `langgraph` dependency. Roadmap milestone M1; see [docs/ROADMAP.md](docs/ROADMAP.md).
- Model router / LLM fabric (`backend/app/services/llm/`) — provider-neutral, policy-driven role-based model selection: a `ModelProvider` protocol returning schema-validated structured output, a `ModelRouter` resolving `ModelRole` → `BoundModel` via a config-sourced `default_policy`, and a hermetic `FakeProvider` for tests. Per-role model ids added to `Settings`. Concrete provider adapters are batched into the network-gated M-LP milestone. Decisions in [ADR 0003](docs/adrs/0003-model-router-llm-fabric.md). Roadmap milestone M2.
- Research Planner agent (`backend/app/agents/research_planner.py`) — the first real reasoning node: `ResearchPlannerAgent` calls the model fabric (`PLANNING` role), maps a model-output DTO into the canonical `ResearchPlan`/`SubQuestion` schema (ids/timestamps minted by the schema, never the model), and raises `PlannerError` on an empty plan. Wired into the workflow `plan` node via factory-closure dependency injection ([ADR 0004](docs/adrs/0004-node-dependency-injection.md)); `run_research(state, *, planner)` now injects dependencies. Fully verified offline via `FakeProvider`. Roadmap milestone M3.
- Workflow error handling + conditional routing (`backend/app/workflows/deep_research.py`) — a deterministic failure path: a uniform `_with_failure_handling` wrapper converts any node exception (e.g. `PlannerError`) into a `FAILED` state update, and the graph's first **conditional edges** route off the typed `status` channel to a terminal `failed` sink. Adds a nullable `error: str | None` field to `ResearchState`. Resolves the agent-vs-deterministic orchestrator question (M4 is deterministic; judgment/quality-gates are deferred to M10). Decisions and deferrals in [ADR 0005](docs/adrs/0005-workflow-error-handling.md). Roadmap milestone M4 (failure path).
- Source Discovery agent + search fabric (`backend/app/agents/source_discovery.py`, `backend/app/services/search/`) — opens the Knowledge Acquisition band. `SourceDiscoveryAgent` plans search queries via the model (`PLANNING` role) and retrieves `Source`s through an injected `SearchProvider` tool (provider-neutral protocol + hermetic `FakeSearchProvider`; concrete network adapter → M-LP). The LLM authors only queries, never a `Source.url` — keeping the evidence-vs-inference boundary (CLAUDE.md §11) structural. Adds a typed `Source.discovered_via` provenance field (symmetric with `Evidence.extracted_via`). Wired into the `acquire` node via factory-closure DI; `run_research(state, *, planner, discovery)`. Single-node acquisition keeps the fan-out reducer deferred to M7. Decisions in [ADR 0006](docs/adrs/0006-source-discovery-and-search-fabric.md). Roadmap milestone M5.
- OpenAI-compatible LLM provider adapter (`backend/app/services/llm/openai_compatible.py`) — the first concrete `ModelProvider`, making real LLM calls possible. An `httpx`-based adapter speaking the OpenAI `/chat/completions` API, so one adapter serves any compatible free backend (Groq, OpenRouter, Together, Cerebras, local Ollama) by configuration alone (`base_url` + `api_key` + `model`; CLAUDE.md §6). Structured output via JSON-object mode + JSON-schema-in-prompt + `model_validate_json` + one error-fed repair retry. Adds `build_router_from_settings` (composition root), a `python -m app.cli.plan "<topic>"` harness, `Settings.base_url`/`api_key` (SecretStr) + `.env` loading, and `.env.example`; promotes `httpx` and `python-dotenv` to runtime dependencies. Fully offline-verifiable via `httpx.MockTransport`; live path covered by a `@pytest.mark.integration` smoke test. **The Research Planner is now runnable end-to-end against a real free LLM.** Decisions in [ADR 0007](docs/adrs/0007-openai-compatible-llm-adapter.md). Roadmap milestone M-LP.1.
- Source Ingestion (`backend/app/services/ingestion/`, M6) — turns discovered `Source`s into `Chunk`s. A `FetchProvider` fabric (hermetic `FakeFetchProvider` + a real, hardened `HttpxFetchProvider` with timeout/size/redirect caps, a content-type allowlist, and no credentials), a pure stdlib HTML parser + deterministic fixed-window chunker, and an `IngestionService` (deterministic tool, CLAUDE.md §4) that ingests web sources only (skipping non-web and per-source failures; raising only on zero chunks). Wired into a new `ingest` node — topology is now `plan → acquire → ingest → reason → publish` — inheriting M4's failure routing. WEB-only v1; PDF/YouTube/OCR (Azure Document Intelligence, Nvidia) and `Chunk.parsed_via` deferred until a multi-parser exists. No schema change. Decisions in [ADR 0008](docs/adrs/0008-source-ingestion-and-fetch-fabric.md). Roadmap milestone M6.
- Evidence Extraction agent (`backend/app/agents/evidence_extraction.py`, M7) — turns ingested `Chunk`s into grounded `Evidence`. An *agent* (`EvidenceExtractionAgent`, `EXTRACTION` model role): it reads each chunk in isolation and extracts the factual claims that chunk supports, each with a confidence. The CLAUDE.md §11 evidence-vs-inference boundary is made **structural** — the model authors only `claim` + `confidence`, while all provenance (`source_id`/`source_url`/`chunk_id`/`chunk_text`) is *code-attached* from the real `Chunk`/`Source`, so a claim can never be misattributed. Tolerates per-chunk model failures (skip + log), raises `ExtractionError` on an unknown source or zero total evidence. Wired into a new `extract` node — topology is now `plan → acquire → ingest → extract → reason → publish` — inheriting M4's failure routing. Introduces the `ResearchDeps` dependency container (the kwarg-threshold trigger pre-registered in ADR 0008); `run_research`/`build_research_graph` now take a single `deps: ResearchDeps`. Fan-out reducer + per-chunk concurrency stay deferred to the checkpointer milestone. No schema change (`Evidence` already exists). Decisions in [ADR 0009](docs/adrs/0009-evidence-extraction.md). Roadmap milestone M7.
- Cross-Verification agent + claim-blocking tool (`backend/app/agents/cross_verification.py`, `backend/app/services/reasoning/`, M8) — opens the Knowledge Reasoning band: turns chunk-local `Evidence` into cross-checked `Verdict`s (corroborated / single-source / contradicted). The agent/tool split (CLAUDE.md §4): a deterministic stdlib `build_claim_blocks` *tool* groups claims sharing salient tokens into candidate clusters (bounding the naive O(N²) cross-product), and the `CrossVerificationAgent` (`PLANNING` role) judges each cluster. The §11 evidence-vs-inference boundary is made **structural** twice: the model references evidence only by *local index* (code resolves + validates ids, dropping any it invents), and `CORROBORATED` requires **≥2 distinct sources** — a count code-derives from the supporting evidence, never trusts to the model (intra-source repetition is downgraded to single-source). Adds a new `KnowledgeReasoningState` substate (`reasoning.verdicts`) on `ResearchState`, plus `SupportLevel` and `Verdict` schema (verdicts reference evidence by id — no inline snapshot; strength is the separate `confidence` float). Replaces the stub `reason` node with a real `verify` node — topology is now `plan → acquire → ingest → extract → verify → publish` — inheriting M4's failure routing; `CrossVerificationAgent` is the 5th `ResearchDeps` collaborator. Thin/single-source support is a valid result (not a failure); raises only on empty input or zero output. Per-cluster concurrency + fan-out reducer deferred to the checkpointer milestone. Decisions in [ADR 0010](docs/adrs/0010-cross-verification.md). Roadmap milestone M8.
- Synthesis agent (`backend/app/agents/synthesis.py`, M9) — the synthesis step of the Knowledge Reasoning band: composes the cross-checked `Verdict`s (+ the plan's sub-questions) into plan-anchored `Finding`s. A single model call over the already-reduced verdict set (`SynthesisAgent`, `LONG_CONTEXT` role — CLAUDE.md §6 "long-context summarization") — a pure agent with no deterministic tool (unlike M8, there is no O(N²) explosion to bound). The §11 boundary is made structural one layer up: the model authors prose + *local indices* into two separate numbered spaces (verdicts `V#`, sub-questions `S#`), and code resolves + validates each against its own list (out-of-range dropped, a finding with no resolvable verdict dropped). The **keystone guard**: the grounding summary — `disputed` (any cited verdict contradicted) and `weakest_support` (floor over the cited verdicts' support levels) — is **code-derived**, with the model given no field to self-report it, so a finding can never overstate its grounding and the caveat travels to the report non-omittably. Adds `Finding` + `Synthesis` schema and a `reasoning.synthesis` substate; the narrative layer (summary/takeaways) is deferred to the publishing band (M11/M12). Replaces no node — adds a `synthesize` node between `verify` and `publish` (topology now `plan → acquire → ingest → extract → verify → synthesize → publish`), inheriting M4's failure routing; `SynthesisAgent` is the 6th `ResearchDeps` collaborator. Thin/disputed synthesis is a valid result; raises only on empty input or zero findings. Map-reduce + fan-out reducer deferred to the checkpointer milestone. Decisions in [ADR 0011](docs/adrs/0011-synthesis.md). Roadmap milestone M9.
