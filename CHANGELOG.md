# Changelog

All notable changes to Reel Automation are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Entries under `[Unreleased]` describe changes that have landed on `main` but
are not yet part of a tagged release. When cutting a release, move the
`[Unreleased]` section under a new version heading with the release date.

## [Unreleased]

### Added

- Performance / benchmark harness (`backend/tests/perf/`) — a stdlib-only (`time.perf_counter`, deliberately not `pytest-benchmark` to avoid a sibling `pyproject` conflict) timing harness plus two offline benchmarks for the hermetic Deep Research pipeline: end-to-end `run_research` latency (real compiled LangGraph graph, `FakeProvider`-backed, model latency factored out) and `build_claim_blocks` scaling across input sizes (surfacing the M8 O(N²)-bounded co-occurrence path). A small `harness` *tool* (CLAUDE.md §4: `time_callable` best/mean/worst over repeats + a fixed-width `render_table`) backs both, and a local `conftest` registers a `perf` marker and prints the timing tables in the run summary (visible even under `-q`). Informational, **not** pass/fail gates — no latency thresholds, only structural sanity asserts. Dual-marked `@pytest.mark.integration` (rides the existing `addopts = "-m 'not integration'"` so they stay out of the default `pytest -q` suite — `pyproject.toml` is out of scope here) and `@pytest.mark.perf` (the positive selector: `pytest -m perf` runs only the benchmarks, excluding the network-gated live tests). Self-contained fakes mirror `tests/integration/test_pipeline_e2e.py`; no new dependencies, no app/config/test changes.
- Media pipeline — the Deep Research → Media handoff orchestrator (`backend/app/media/pipeline.py`) — a deterministic `MediaPipeline` *tool* (no LLM, CLAUDE.md §3.3/§4) that turns a `CreatorPacket` into a `MediaPlan` (assembled-video descriptor) by chaining the existing media seams: select a `NarrativeOption` → synthesize narration once via the injected `TTSProvider` → allocate caption timings across beats → build a `CaptionTrack` via `DeterministicSubtitleService` → compose via the injected `CompositionService`. Fulfils the handoff contract ADR 0019 deferred. DI + skip/raise mirror `IngestionService` (TTS/composition required, subtitles default to the real service; tolerate blank beats, raise `MediaPipelineError` only when no narratable beat survives). The single, deliberate ADR 0019 §4 coupling exception (this file imports `CreatorPacket`; the seam modules stay decoupled). Timing is synthesize-once + **cumulative integer boundaries**, holding the invariant `cues[-1].end_ms == audio.duration_ms == video.duration_ms` exactly. `visual_uris` pass through (visual sourcing stays deferred). Fully hermetic via the fakes. Decisions in [ADR 0025](docs/adrs/0025-media-pipeline.md). Media Production Layer.
- Named provider registry for OpenAI-compatible backends (`backend/app/services/llm/providers.py`) — a `name → ProviderPreset(base_url, key-reader)` registry (`groq`, `nvidia`, `huggingface`, local `ollama`) plus `build_provider(name, settings) -> OpenAICompatibleProvider`. The registry owns each preset's `base_url`; the operator selects a backend by name and supplies only the key, so several providers' keys coexist in one `.env` and switching backend is a name change, not a URL+key edit. Builds the one ADR 0007 adapter (no new adapter); additive alongside `factory.py` (the generic default slot) — does not touch routing policy, `router.py`, or wiring. Adds three namespaced `Settings` keys (`groq_api_key`/`nvidia_api_key`/`huggingface_api_key`, SecretStr) + an `.env.example` block; Ollama is keyless. Fully hermetic-tested (registry resolution, key wiring, keyless-local path, typed unknown-name error). Decisions in [ADR 0028](docs/adrs/0028-provider-registry.md). Roadmap milestone M-LP.
- FFmpeg composition adapter (`backend/app/media/composition/ffmpeg.py`) — the first concrete `CompositionService` behind the ADR 0019 seam, filling the real-ffmpeg slot that ADR deferred. A deterministic *tool* (CLAUDE.md §4 — composition/FFmpeg assembly, no LLM) that assembles narration audio + a structured `CaptionTrack` + ordered visuals into a vertical short-form MP4. The load-bearing design (ADR 0023): **command construction is a pure function** (`build_ffmpeg_args`, taking already-resolved local paths → argv list, no temp files / no random ids / no I/O) split from **execution** (the async `render`'s single mockable `subprocess.run` seam, run off the event loop via `asyncio.to_thread`), so the exact argv is unit-testable token-by-token with **no ffmpeg binary present**. URI→path resolution is explicit (`file://` or bare path; other schemes raise `CompositionError`) — never silently handed to ffmpeg. Missing binary and non-zero exit both normalize to one `CompositionError` carrying the `shlex.join`'d command + a stderr tail (execution always uses the argv list, never a shell). Duration mirrors the narration (no second `ffprobe` binary); captions are burned in via a sidecar `.srt` rendered with the in-layer `subtitles.base.format_srt` (reused, not reimplemented); `produced_via="composition:ffmpeg"`. Preserves the `FakeCompositionService` contract (duration/dimension echo, `RecordedRender` call-capture) so fake↔real swap freely. No new dependency (stdlib `subprocess`/`shlex` + existing Pydantic DTOs); no wiring/composition-root change (adapter only). Argv-construction, URI-resolution, and subprocess-failure paths are fully hermetic; real rendering is a `@pytest.mark.integration` test that synthesizes inputs via ffmpeg lavfi and skips when the binary is absent. Decisions in [ADR 0023](docs/adrs/0023-ffmpeg-composition.md). Media Production Layer.
- Visual / B-roll retrieval seam (`backend/app/media/visuals/`) — the retrieval half of the Media Production layer's "image/video generation or retrieval" responsibility (CLAUDE.md §3.3), the seam ADR 0019 deferred until a composition consumer existed (`CompositionService.render(..., visual_uris=...)` now has a producer for its B-roll uris). A deterministic *tool* (CLAUDE.md §4 — the upstream Short-Form Content Strategist decides *what* the B-roll depicts; this band *executes* the retrieval), mirroring the search fabric point-for-point: a `VisualProvider` protocol + a `VisualClip` DTO (id-prefixed `vis_`, `extra='forbid'`, required `produced_via` provenance `"visuals:fake"`/`"visuals:stock"`; load-bearing `kind`/`width`/`height`/optional `duration_ms` (None for stills)/optional `attribution` for the B-roll-synced-to-script use case) + a hermetic `FakeVisualProvider` (factory-style, records calls) — all self-contained in `base.py` (where `SearchResult` lives beside its protocol; `_gen_id` is the ADR 0019-blessed local copy, not a cross-layer import). The provider — never an LLM — mints the asset `uri`, keeping the retrieval-vs-inference boundary (CLAUDE.md §11) structural. Adds a real httpx-based `StockVisualProvider` over the Pexels Videos API (`GET /videos/search`, `Authorization` auth, portrait orientation, `per_page` clamped to 80, seconds→ms), hardened like the Brave adapter (ADR 0021): empty key raises, key never leaks into repr/errors, injectable client + bounded timeout; operational failures (429/timeout/5xx) propagate as `httpx` errors while only a malformed response *shape* is wrapped in a local `VisualError` (empty result set is valid). The key is taken at construction (not `Settings`), keeping the seam config-root-agnostic; the `@pytest.mark.integration` live smoke test reads `REEL_AUTOMATION_STOCK_API_KEY` from the environment. Fully offline-verifiable via `httpx.MockTransport`. No wiring/composition root and no `config.py`/`schemas.py` change (adapter-only, per scope). `httpx` is already a runtime dependency. Decisions in [ADR 0024](docs/adrs/0024-visual-retrieval.md). Media Production Layer (concrete adapters).
- LLM response cache (`backend/app/services/llm/cache.py`) — a `CachingModelProvider` that **wraps** any `ModelProvider` (decorator pattern — composition, not inheritance) and memoizes `complete_structured` results, keyed by a stable SHA-256 over `(model, system, prompt, schema-identity)`. Schema identity is *both* the qualified class name and the full JSON Schema, so distinct schemas (even with identical fields) never collide. Storage is pluggable via a tiny `CacheBackend` protocol with an in-memory default (`InMemoryCacheBackend`, stdlib `OrderedDict`, optional `max_size` LRU). A hit skips the wrapped call entirely; a miss calls through and populates. Returned values are deep-copied (`model_copy`) on store and return so a caller cannot corrupt a cached entry; **exceptions are never cached** (a failed call leaves the key absent and the next identical call retries). Opt-in by design — wrapping a non-deterministic model trades freshness for cost/latency, so nothing wraps a provider by default (caveat documented in the module). Stdlib only (`hashlib`/`json`); no new dependency, no router/config change. Fully hermetic tests over a `FakeProvider` (called once across two identical requests, not at all on the second; distinct-prompt / distinct-schema / LRU-eviction / exception-not-cached / value-isolation cases). Decisions in [ADR 0026](docs/adrs/0026-llm-response-cache.md).
- Property-based tests for the Deep Research state/provenance schema (`backend/tests/schemas/test_research_state_properties.py`) — `hypothesis` strategies that assert the schema's *invariants* across many generated inputs, complementing the example-based `test_research_state.py`: dict-mode round-trip (`model_validate(model_dump()) == m`) for every model, `_gen_id` prefix + 16-hex-suffix + uniqueness, `extra='forbid'` rejection of unknown keys, `Evidence`/`Verdict` confidence bounded to `[0, 1]`, and tz-aware default timestamps. Explicit bounded `@st.composite` strategies (no `st.builds`/`from_type` — the schema's `from __future__ import annotations` defeats type inference) in a single model→strategy registry, parametrized so a new model is one registry line from full coverage; deterministic (`deadline=None`, `max_examples=50`, UTC-pinned datetimes). Adds the `hypothesis` dev dependency; the module `importorskip`s it so the suite stays green in the offline build sandbox (no PyPI) and runs the properties wherever it is installed (CI, local dev). Tests-only; no schema or app change.
- HTTP TTS provider adapter (`backend/app/media/tts/http_tts.py`) — the first concrete `TTSProvider` behind the ADR 0019 media seam, the media-layer analogue of the M-LP.1 LLM adapter. An `httpx`-based adapter over a *generic* REST TTS endpoint (`POST {base_url}/synthesize`, `Authorization: Bearer`, JSON `{text, voice}` → raw audio bytes), so one adapter serves any backend exposing the contract by configuration alone (CLAUDE.md §6). Because the fixed protocol seam (`synthesize(*, text, voice) -> SynthesizedSpeech`) returns a *descriptor* and `SynthesizedSpeech` is "not the audio bytes … owned by storage" (ADR 0019), the bytes→`audio_uri` step is an injected `AudioSink = Callable[[bytes], str]` (mirroring the injected `client`) — the adapter never embeds bytes or chooses storage. `duration_ms` (required, unrecoverable from opaque bytes without a format parser) comes from an `X-Audio-Duration-Ms` response header; a missing/non-integer/negative header raises `HttpTtsError` rather than silently propagating a wrong `0` into composition timing. Hardened like the LLM adapter: bounded timeout, key-at-construction (so `config.py` is untouched, no key leak), `raise_for_status` (operational failures propagate as `httpx` errors; only bad response *shape* is wrapped in the locally-defined `HttpTtsError`). Fully offline-verifiable via `httpx.MockTransport` + a capturing fake sink; live path covered by a `@pytest.mark.integration` smoke test (skipped without `REEL_TTS_BASE_URL`/`REEL_TTS_API_KEY`). Adapter only — no factory/router wiring, no `Settings` field, no schema change, no new dependency (`httpx` is already runtime). Decisions in [ADR 0022](docs/adrs/0022-tts-adapter.md). Media Production layer (concrete adapters).

Decisions in [ADR 0030](docs/adrs/0030-structured-logging.md). Observability scaffold (CLAUDE.md §3.1).

Citations and caveats **always render** when non-empty — the §11 non-omittability of provenance and caveats carried to the output surface; the renderer fabricates nothing beyond the `Report`. `render_html` escapes all model/user text (`html.escape`, `quote=True` for the `href` attribute context). - Evidence Extraction agent (`backend/app/agents/evidence_extraction.py`, M7) — turns ingested `Chunk`s into grounded `Evidence`. An *agent* (`EvidenceExtractionAgent`, `EXTRACTION` model role): it reads each chunk in isolation and extracts the factual claims that chunk supports, each with a confidence. The CLAUDE.md §11 evidence-vs-inference boundary is made **structural** — the model authors only `claim` + `confidence`, while all provenance (`source_id`/`source_url`/`chunk_id`/`chunk_text`) is *code-attached* from the real `Chunk`/`Source`, so a claim can never be misattributed. Tolerates per-chunk model failures (skip + log), raises `ExtractionError` on an unknown source or zero total evidence. Wired into a new `extract` node — topology is now `plan → acquire → ingest → extract → reason → publish` — inheriting M4's failure routing. Introduces the `ResearchDeps` dependency container (the kwarg-threshold trigger pre-registered in ADR 0008); `run_research`/`build_research_graph` now take a single `deps: ResearchDeps`. Roadmap milestone M7.
- Cross-Verification agent + claim-blocking tool (`backend/app/agents/cross_verification.py`, `backend/app/services/reasoning/`, M8) — opens the Knowledge Reasoning band: turns chunk-local `Evidence` into cross-checked `Verdict`s (corroborated / single-source / contradicted). The agent/tool split (CLAUDE.md §4): a deterministic stdlib `build_claim_blocks` *tool* groups claims sharing salient tokens into candidate clusters (bounding the naive O(N²) cross-product), and the `CrossVerificationAgent` (`PLANNING` role) judges each cluster. The §11 evidence-vs-inference boundary is made **structural** twice: the model references evidence only by *local index* (code resolves + validates ids, dropping any it invents), and `CORROBORATED` requires **≥2 distinct sources** — a count code-derives from the supporting evidence, never trusts to the model (intra-source repetition is downgraded to single-source). Adds a new `KnowledgeReasoningState` substate (`reasoning.verdicts`) on `ResearchState`, plus `SupportLevel` and `Verdict` schema (verdicts reference evidence by id — no inline snapshot; strength is the separate `confidence` float). Replaces the stub `reason` node with a real `verify` node — topology is now `plan → acquire → ingest → extract → verify → publish` — inheriting M4's failure routing; `CrossVerificationAgent` is the 5th `ResearchDeps` collaborator. Thin/single-source support is a valid result (not a failure); raises only on empty input or zero output. Per-cluster concurrency + fan-out reducer deferred to the checkpointer milestone. Decisions in [ADR 0010](docs/adrs/0010-cross-verification.md). Roadmap milestone M8.
- Synthesis agent (`backend/app/agents/synthesis.py`, M9) — the synthesis step of the Knowledge Reasoning band: composes the cross-checked `Verdict`s (+ the plan's sub-questions) into plan-anchored `Finding`s. A single model call over the already-reduced verdict set (`SynthesisAgent`, `LONG_CONTEXT` role — CLAUDE.md §6 "long-context summarization") — a pure agent with no deterministic tool (unlike M8, there is no O(N²) explosion to bound). The §11 boundary is made structural one layer up: the model authors prose + *local indices* into two separate numbered spaces (verdicts `V#`, sub-questions `S#`), and code resolves + validates each against its own list (out-of-range dropped, a finding with no resolvable verdict dropped). The **keystone guard**: the grounding summary — `disputed` (any cited verdict contradicted) and `weakest_support` (floor over the cited verdicts' support levels) — is **code-derived**, with the model given no field to self-report it, so a finding can never overstate its grounding and the caveat travels to the report non-omittably. Adds `Finding` + `Synthesis` schema and a `reasoning.synthesis` substate; the narrative layer (summary/takeaways) is deferred to the publishing band (M11/M12). Replaces no node — adds a `synthesize` node between `verify` and `publish` (topology now `plan → acquire → ingest → extract → verify → synthesize → publish`), inheriting M4's failure routing; `SynthesisAgent` is the 6th `ResearchDeps` collaborator. Thin/disputed synthesis is a valid result; raises only on empty input or zero findings. Map-reduce + fan-out reducer deferred to the checkpointer milestone. Decisions in [ADR 0011](docs/adrs/0011-synthesis.md). Roadmap milestone M9.
- Editorial Critic (`backend/app/agents/editorial_critic.py`, `backend/app/services/reasoning/coverage.py`, M10a) — closes the Knowledge Reasoning band: assesses the `Synthesis` for coverage and quality, realizing the quality-gate logic ADR 0005 deferred to M10. The agent/tool split (CLAUDE.md §4): a deterministic stdlib `coverage` tool computes which sub-questions are addressed by zero findings, and the `EditorialCriticAgent` (`PLANNING` role) judges what code cannot (redundancy, balance, clarity, overstated-vs-disputed prose). The §11 boundary is made structural one layer up: the model references findings/sub-questions only by *local index* (two separate `F#`/`S#` spaces, code-resolved + validated, issues about nothing dropped), and the accept/revise `decision` + coverage gap are **code-derived** — the model gets no field to vote, so it cannot ACCEPT past an objective coverage gap. A disputed/single-source finding is *not* a revise trigger (a valid surfaced outcome per ADR 0010/0011); only coverage gaps and quality issues trigger revise. Adds `CritiqueDecision` / `QualityIssueKind` / `QualityIssue` / `Critique` schema and a `reasoning.critiques` substate. Adds a `critique` node — topology `plan → acquire → ingest → extract → verify → synthesize → critique → publish`, still **linear**: the `decision` is *recorded, not yet routed on* (the bounded revision loop that consumes it — first graph cycle, iteration cap, feed-forward — is the separately-gated M10b). `EditorialCriticAgent` is the 7th `ResearchDeps` collaborator. "Found nothing wrong" is a valid ACCEPT, not a failure; raises only on an empty synthesis. Decisions in [ADR 0012](docs/adrs/0012-editorial-critic.md). Roadmap milestone M10a.
- Revision loop (`backend/app/workflows/deep_research.py`, `backend/app/agents/synthesis.py`, M10b) — the bounded `critique → synthesize` back-edge, the graph's first **cycle**. The Editorial Critic's `decision` is now routed on: `_make_critique_router` sends `revise` back to `synthesize`, `accept`/`exhausted` to `publish`, `failed` to the sink. The **router, not the critic agent, owns termination** — a code-incremented top-level `revision_iteration` counter (single-writer: the critique node; placed top-level so the synthesize node's `reasoning` rewrite on the back-edge cannot re-zero it) is compared against a `max_syntheses` cap, forcing `exhausted` once reached regardless of the model's verdict (model proposes revise, code decides — the M8/M9 shape on the continue/stop axis). Re-synthesis feeds the prior critique forward (`prior_critique` param on `SynthesisAgent.synthesize`, additive/backward-compatible) so the loop *addresses* the critique rather than re-running identical inputs. A revision-exhausted run **completes** with a best-effort synthesis (not a failure — the "thin result is valid" inversion applied to the loop); exhausted-vs-clean-accept is derivable from `revision_iteration == max_syntheses` with the last critique still `REVISE`. An explicit `recursion_limit` on `ainvoke` backstops the cycle above its legitimate worst case (a hit raises an uncatchable `GraphRecursionError`, so the code cap must always fire first). Adds one top-level `revision_iteration` field; no change to the M10a `Critique`/reasoning schema. Decisions in [ADR 0012](docs/adrs/0012-editorial-critic.md). Roadmap milestone M10b.
- Deep Research API surface (`backend/app/api/research.py`, `backend/app/api/deps.py`, `backend/app/services/composition.py`, M13 partial) — the first HTTP entry point to the pipeline. A **thin** router (CLAUDE.md §10): `POST /api/v1/research` validates a `ResearchJobRequest` (`topic` + bounded `max_syntheses`), awaits `run_research`, and returns the canonical terminal `ResearchState` as the response model.
