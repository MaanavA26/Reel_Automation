# Changelog

All notable changes to Reel Automation are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

are not yet part of a tagged release. When cutting a release, move the

### Added

- FFmpeg composition adapter (`backend/app/media/composition/ffmpeg.py`) тАФ the first concrete `CompositionService` behind the ADR 0019 seam, filling the real-ffmpeg slot that ADR deferred. A deterministic *tool* (CLAUDE.md ┬з4 тАФ composition/FFmpeg assembly, no LLM) that assembles narration audio + a structured `CaptionTrack` + ordered visuals into a vertical short-form MP4. The load-bearing design (ADR 0023): **command construction is a pure function** (`build_ffmpeg_args`, taking already-resolved local paths тЖТ argv list, no temp files / no random ids / no I/O) split from **execution** (the async `render`'s single mockable `subprocess.run` seam, run off the event loop via `asyncio.to_thread`), so the exact argv is unit-testable token-by-token with **no ffmpeg binary present**. URIтЖТpath resolution is explicit (`file://` or bare path; other schemes raise `CompositionError`) тАФ never silently handed to ffmpeg. Missing binary and non-zero exit both normalize to one `CompositionError` carrying the `shlex.join`'d command + a stderr tail (execution always uses the argv list, never a shell). Duration mirrors the narration (no second `ffprobe` binary); captions are burned in via a sidecar `.srt` rendered with the in-layer `subtitles.base.format_srt` (reused, not reimplemented); `produced_via="composition:ffmpeg"`. Preserves the `FakeCompositionService` contract (duration/dimension echo, `RecordedRender` call-capture) so fakeтЖФreal swap freely. No new dependency (stdlib `subprocess`/`shlex` + existing Pydantic DTOs); no wiring/composition-root change (adapter only). Argv-construction, URI-resolution, and subprocess-failure paths are fully hermetic; real rendering is a `@pytest.mark.integration` test that synthesizes inputs via ffmpeg lavfi and skips when the binary is absent. Decisions in [ADR 0023](docs/adrs/0023-ffmpeg-composition.md). Media Production Layer.

- Evidence Extraction agent (`backend/app/agents/evidence_extraction.py`, M7) тАФ turns ingested `Chunk`s into grounded `Evidence`. Roadmap milestone M7.
- Cross-Verification agent + claim-blocking tool (`backend/app/agents/cross_verification.py`, `backend/app/services/reasoning/`, M8) тАФ opens the Knowledge Reasoning band: turns chunk-local `Evidence` into cross-checked `Verdict`s (corroborated / single-source / contradicted). The agent/tool split (CLAUDE.md ┬з4): a deterministic stdlib `build_claim_blocks` *tool* groups claims sharing salient tokens into candidate clusters (bounding the naive O(N┬▓) cross-product), and the `CrossVerificationAgent` (`PLANNING` role) judges each cluster. The ┬з11 evidence-vs-inference boundary is made **structural** twice: the model references evidence only by *local index* (code resolves + validates ids, dropping any it invents), and `CORROBORATED` requires **тЙе2 distinct sources** тАФ a count code-derives from the supporting evidence, never trusts to the model (intra-source repetition is downgraded to single-source). Adds a new `KnowledgeReasoningState` substate (`reasoning.verdicts`) on `ResearchState`, plus `SupportLevel` and `Verdict` schema (verdicts reference evidence by id тАФ no inline snapshot; strength is the separate `confidence` float). Replaces the stub `reason` node with a real `verify` node тАФ topology is now `plan тЖТ acquire тЖТ ingest тЖТ extract тЖТ verify тЖТ publish` тАФ inheriting M4's failure routing; `CrossVerificationAgent` is the 5th `ResearchDeps` collaborator. Thin/single-source support is a valid result (not a failure); raises only on empty input or zero output. Per-cluster concurrency + fan-out reducer deferred to the checkpointer milestone. Decisions in [ADR 0010](docs/adrs/0010-cross-verification.md). Roadmap milestone M8.
- Synthesis agent (`backend/app/agents/synthesis.py`, M9) тАФ the synthesis step of the Knowledge Reasoning band: composes the cross-checked `Verdict`s (+ the plan's sub-questions) into plan-anchored `Finding`s. A single model call over the already-reduced verdict set (`SynthesisAgent`, `LONG_CONTEXT` role тАФ CLAUDE.md ┬з6 "long-context summarization") тАФ a pure agent with no deterministic tool (unlike M8, there is no O(N┬▓) explosion to bound). The ┬з11 boundary is made structural one layer up: the model authors prose + *local indices* into two separate numbered spaces (verdicts `V#`, sub-questions `S#`), and code resolves + validates each against its own list (out-of-range dropped, a finding with no resolvable verdict dropped). The **keystone guard**: the grounding summary тАФ `disputed` (any cited verdict contradicted) and `weakest_support` (floor over the cited verdicts' support levels) тАФ is **code-derived**, with the model given no field to self-report it, so a finding can never overstate its grounding and the caveat travels to the report non-omittably. Adds `Finding` + `Synthesis` schema and a `reasoning.synthesis` substate; the narrative layer (summary/takeaways) is deferred to the publishing band (M11/M12). Replaces no node тАФ adds a `synthesize` node between `verify` and `publish` (topology now `plan тЖТ acquire тЖТ ingest тЖТ extract тЖТ verify тЖТ synthesize тЖТ publish`), inheriting M4's failure routing; `SynthesisAgent` is the 6th `ResearchDeps` collaborator. Thin/disputed synthesis is a valid result; raises only on empty input or zero findings. Map-reduce + fan-out reducer deferred to the checkpointer milestone. Decisions in [ADR 0011](docs/adrs/0011-synthesis.md). Roadmap milestone M9.
- Editorial Critic (`backend/app/agents/editorial_critic.py`, `backend/app/services/reasoning/coverage.py`, M10a) тАФ closes the Knowledge Reasoning band: assesses the `Synthesis` for coverage and quality, realizing the quality-gate logic ADR 0005 deferred to M10. The agent/tool split (CLAUDE.md ┬з4): a deterministic stdlib `coverage` tool computes which sub-questions are addressed by zero findings, and the `EditorialCriticAgent` (`PLANNING` role) judges what code cannot (redundancy, balance, clarity, overstated-vs-disputed prose). The ┬з11 boundary is made structural one layer up: the model references findings/sub-questions only by *local index* (two separate `F#`/`S#` spaces, code-resolved + validated, issues about nothing dropped), and the accept/revise `decision` + coverage gap are **code-derived** тАФ the model gets no field to vote, so it cannot ACCEPT past an objective coverage gap. A disputed/single-source finding is *not* a revise trigger (a valid surfaced outcome per ADR 0010/0011); only coverage gaps and quality issues trigger revise. Adds `CritiqueDecision` / `QualityIssueKind` / `QualityIssue` / `Critique` schema and a `reasoning.critiques` substate. Adds a `critique` node тАФ topology `plan тЖТ acquire тЖТ ingest тЖТ extract тЖТ verify тЖТ synthesize тЖТ critique тЖТ publish`, still **linear**: the `decision` is *recorded, not yet routed on* (the bounded revision loop that consumes it тАФ first graph cycle, iteration cap, feed-forward тАФ is the separately-gated M10b). `EditorialCriticAgent` is the 7th `ResearchDeps` collaborator. "Found nothing wrong" is a valid ACCEPT, not a failure; raises only on an empty synthesis. Decisions in [ADR 0012](docs/adrs/0012-editorial-critic.md). Roadmap milestone M10a.
- Revision loop (`backend/app/workflows/deep_research.py`, `backend/app/agents/synthesis.py`, M10b) тАФ the bounded `critique тЖТ synthesize` back-edge, the graph's first **cycle**. The Editorial Critic's `decision` is now routed on: `_make_critique_router` sends `revise` back to `synthesize`, `accept`/`exhausted` to `publish`, `failed` to the sink. The **router, not the critic agent, owns termination** тАФ a code-incremented top-level `revision_iteration` counter (single-writer: the critique node; placed top-level so the synthesize node's `reasoning` rewrite on the back-edge cannot re-zero it) is compared against a `max_syntheses` cap, forcing `exhausted` once reached regardless of the model's verdict (model proposes revise, code decides тАФ the M8/M9 shape on the continue/stop axis). Re-synthesis feeds the prior critique forward (`prior_critique` param on `SynthesisAgent.synthesize`, additive/backward-compatible) so the loop *addresses* the critique rather than re-running identical inputs. A revision-exhausted run **completes** with a best-effort synthesis (not a failure тАФ the "thin result is valid" inversion applied to the loop); exhausted-vs-clean-accept is derivable from `revision_iteration == max_syntheses` with the last critique still `REVISE`. An explicit `recursion_limit` on `ainvoke` backstops the cycle above its legitimate worst case (a hit raises an uncatchable `GraphRecursionError`, so the code cap must always fire first). Adds one top-level `revision_iteration` field; no change to the M10a `Critique`/reasoning schema. Decisions in [ADR 0012](docs/adrs/0012-editorial-critic.md). Roadmap milestone M10b.

- Brave Search provider adapter (`backend/app/services/search/brave_search.py`, M-LP.2) тАФ a second concrete `SearchProvider` for failover/robustness alongside the Tavily adapter (ADR 0013). An `httpx`-based adapter over the Brave Web Search API (`GET /res/v1/web/search`, `X-Subscription-Token` auth), mapping `web.results[]` тЖТ `SearchResult` (web type; Brave's `description` тЖТ snippet; URL-less hits skipped; `count` clamped to Brave's max of 20). The provider тАФ never the LLM тАФ mints the `url`, keeping the evidence-vs-inference boundary (CLAUDE.md ┬з11; ADR 0006) structural. Operational failures (429/timeout/5xx via `raise_for_status`) propagate as `httpx` errors for the Orchestrator; only a malformed response *shape* is wrapped in a locally-defined `SearchError` (an absent/empty `web` block is a valid empty result, not an error). Adds `Settings.brave_api_key` (SecretStr), distinct from the LLM `api_key`. Fully offline-verifiable via `httpx.MockTransport`; live path covered by a `@pytest.mark.integration` smoke test. Per scope discipline only the new adapter module + config field are added тАФ shared `base.py`/`__init__.py` are untouched, so the change stays merge-clean with the sibling Tavily branch. Decisions in [ADR 0021](docs/adrs/0021-brave-search-adapter.md). Roadmap milestone M-LP.2.
- Media Production Layer scaffold (`backend/app/media/`) тАФ opens the second major component (CLAUDE.md ┬з3.3) as provider-neutral, fully-faked *tool/service* seams (CLAUDE.md ┬з4 тАФ media work is deterministic execution, never agents; introduced via ADR per ┬з3.4/┬з16). Three seams mirror the LLM/search fabric pattern: a `TTSProvider` protocol + hermetic `FakeTTSProvider` (text тЖТ `SynthesizedSpeech`), a `CompositionService` protocol + hermetic `FakeCompositionService` wrapping the future FFmpeg step (assets тЖТ `RenderedVideo`; **no real ffmpeg invoked**), and тАФ deliberately asymmetric тАФ a subtitle band that ships **real** code now: a synchronous `SubtitleService` protocol, a concrete `DeterministicSubtitleService`, and pure stdlib `format_srt`/`format_vtt` formatters (the band needs no external service). Typed artifact DTOs (`backend/app/media/schemas.py`: `SynthesizedSpeech`/`aud_`, `CaptionTrack`+`Caption`/`sub_`, `RenderedVideo`/`vid_`) are `extra='forbid'`, id-prefixed, and each top-level artifact carries a typed, required `produced_via` provenance string (symmetric with `Source.discovered_via`/`Evidence.extracted_via`; ADR 0006) тАФ extending provenance integrity to the pipeline's tail. The layer imports nothing from `app.schemas`/`app.agents`/`app.workflows` (a documented local `_gen_id` copy keeps it decoupled), so it builds/tests/showcases standalone. Concrete adapters (ElevenLabs/Veo/real ffmpeg) and the Deep Research creator-packet тЖТ media handoff contract are explicitly deferred (the latter earns its own ADR once M12's packet shape is fixed). No new dependency (stdlib + Pydantic). Decisions in [ADR 0019](docs/adrs/0019-media-production-layer.md).
- Gemini-native LLM provider adapter (`backend/app/services/llm/gemini.py`, M-LP.3) тАФ the second concrete `ModelProvider`, whose value over the OpenAI-compatible adapter is **native structured output**: an `httpx`-based adapter over the Gemini `generateContent` REST API that sets `responseSchema` + `responseMimeType: application/json` so decoding is schema-constrained server-side (vs schema-in-prompt). A bounded `_to_gemini_schema` sanitizer inlines `$ref`/`$defs` and strips Gemini-rejected JSON-Schema keys (`title`/`additionalProperties`/`default`), collapsing `Optional[X]` to a single `nullable` branch; `x-goog-api-key` **header** auth keeps the key out of request URLs; one error-fed repair retry mirrors the OpenAI adapter (ADR 0007). Fully offline-verifiable via `httpx.MockTransport` тАФ including the nested-schema sanitization path (the load-bearing guard); live path covered by a `@pytest.mark.integration` smoke test. Router wiring (`factory._build_provider`) is a deliberately deferred one-branch follow-up, keeping this diff scoped to the adapter + settings. - Containerization + deploy CI (`backend/Dockerfile`, `frontend/Dockerfile`, `docker-compose.yml`, per-context `.dockerignore`, `.github/workflows/docker-build.yml`) тАФ packages the FastAPI backend and the Vite/React frontend as deployable images without touching application code. The **backend** image is multi-stage (a `python:3.11-slim` builder installs the app into an isolated `/opt/venv` from `pyproject.toml`; a slim runtime copies only the venv + `app/`), runs as a non-root `appuser`, binds uvicorn `app.main:app` on `0.0.0.0:8000`, and self-checks `/api/v1/health` via a stdlib `urllib` `HEALTHCHECK` (no `curl`, no secrets baked). The **`docker-build.yml`** workflow builds both images (build-only тАФ no registry push) on PR/push to `main`, mirroring `ci.yml` conventions (`permissions: contents: read`, `concurrency` cancel-in-progress, `actions/checkout@v4`, `ubuntu-latest`) via plain `docker build` steps тАФ zero third-party-action or registry-auth surface, trivially actionlint-clean. - Full-pipeline integration test harness (`backend/tests/integration/test_pipeline_e2e.py`, `backend/tests/integration/test_pipeline_live.py`) тАФ a cohesive cross-band E2E suite for `run_research`.
