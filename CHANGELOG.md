## [Unreleased]

### Security

- Fetch + render trust-boundary hardening (`backend/app/services/ingestion/httpx_fetch.py`, `backend/app/services/publishing/html.py`, LLM/search/visual adapters) — five audit findings fixed at the boundaries (no schema retyping). **SSRF (HIGH):** `HttpxFetchProvider` now follows redirects *manually* (`follow_redirects=False`) and re-runs a scheme allowlist (`http`/`https`) + private/loopback/link-local/reserved-IP guard (`ipaddress`, cloud-metadata `169.254.169.254` included, IPv4-mapped IPv6 unwrapped) on the initial URL **and every redirect hop**, via an injectable `resolver` seam so hostname checks stay hermetic. **DoS (MED):** the size cap is enforced by streaming (`client.stream` + `aiter_bytes`, aborted once the running total exceeds `max_bytes`) with an oversized-`Content-Length` pre-reject — the body is never fully buffered. **Content-type bypass (MED):** a missing `Content-Type` header now fails closed (rejected). **XSS (HIGH):** `render_html` validates a citation's `source_url` scheme against `{http, https, mailto}` (whitespace/control chars stripped first) before linking — a `javascript:`/`data:` url renders as escaped text with no `href`. **Info-leak (LOW):** upstream-response excerpts in `openai_compatible`/`gemini`/Brave/Tavily/stock adapter errors are clipped to a bounded prefix so full bodies don't reach `ResearchState.error`/logs. Hermetic tests cover each (redirect-to-loopback/metadata blocked, dangerous-scheme href dropped, oversized stream aborted, missing content-type rejected); no new dependency, no schema change. Decisions in [ADR 0043](docs/adrs/0043-fetch-and-render-hardening.md).

Fully hermetic (pure-argv + mocked-subprocess paths); the `@pytest.mark.integration` real render skips on **two** conditions (ffmpeg *or* font absent) and synthesizes its input via `lavfi`. - Pre-publish content-safety guardrail (`backend/app/safety/`) — a deterministic `PrePublishGate` *tool* (CLAUDE.md §4 — a policy check, no judgment, no LLM) that converts the §11 code-derived caveats/warnings into a single publish decision, the structural "don't auto-post misinformation" guard for a faceless short-form channel. Given a `Report` + `CreatorPacket` + a `PublishCandidate` (the produced script/metadata, the gate's own in-package DTO since it cannot touch `schemas/`), `gate.evaluate(...)` runs every rule and returns a typed `SafetyVerdict` (ALLOW / BLOCK / REVIEW) whose `decision` is the **maximum severity** over an explained `SafetyReason` list — so one pass surfaces every problem. Verb-aligned policy: a `DISPUTED_FINDING` caveat on the report **or** a disputed `CreatorWarning` on the packet → **BLOCK** (content rests on contradictory sources); an `UNRESOLVED_CRITIQUE` caveat (exhausted-revision banner) → **BLOCK** without an explicit `disclaimer` on the candidate, downgraded to **REVIEW** with one; a configurable banned-topic/keyword match → **REVIEW** (case-insensitive, whole-word by default so `"thorpe"` doesn't trip on `"Scunthorpe"`, opt-in substring, severity configurable to BLOCK); and a source-grounding floor over **distinct** `Citation.source_id`s (mirroring the CORROBORATED ≥2-distinct-sources semantics) → **REVIEW** below threshold. Only `DISPUTED_FINDING`/`UNRESOLVED_CRITIQUE` drive the verdict — the benign caveat kinds (`WEAK_SUPPORT`/`UNCOVERED_SUB_QUESTION`/`QUALITY_ISSUE`) are explicitly *not* block signals, so a clean report stays ALLOW-reachable. **Trusts** the upstream non-omittable §11 signals (read-only `CaveatKind` import) rather than re-deriving them — no `Finding`/`Synthesis` coupling. The explainable policy is configured by a frozen `GatePolicy` via **constructor args, not `Settings`** (CLAUDE.md §10 — config-root-agnostic, per-call-site tunable). Pure + fully unit-testable: the `SafetyVerdict` is id-free/timestamp-free (a comparable value object, mirroring `caveats.py`), no LLM/IO/clock. Capability only — the gate is a ready tool, **not yet wired** into the publishing band/endpoint (the M-LP capability-before-wiring pattern); `config.py`/`main.py`/`pyproject.toml`/`app/publishing/` untouched (stdlib + Pydantic only, no new dependency). Fully hermetic tests (clean→ALLOW, disputed report/packet→BLOCK, exhausted-critique with/without disclaimer→BLOCK/REVIEW, banned-keyword whole-word/substring/severity→flag, distinct-source floor→REVIEW, max-severity precedence, value-object equality). Decisions in [ADR 0041](docs/adrs/0041-pre-publish-safety.md). Research Publishing band quality gate (CLAUDE.md §5.5 band A quality gates / §5.4 unsafe-claim warnings).

A deterministic *tool* (CLAUDE.md §4 — API wrapper, no reasoning) wrapping the four real endpoints: `health()` → `GET /health`, `submit_research(topic, *, max_syntheses=None)` → `POST /research` (synchronous, returns the terminal `ResearchState`), `enqueue_job(...)` → `POST /research/jobs` (202, returns the queued job id), and `get_job(id)` → `GET /research/jobs/{id}`. **Stays in lockstep with the server by importing its real models** — requests are built/validated through the router's own `ResearchJobRequest` and responses parsed into the canonical `ResearchState` / `HealthResponse`, so a contract change flows to the client for free. Synchronous + single-client by design (CLAUDE.md §7 — no speculative async surface); usable as a context manager. Top-level `examples/` ships three illustrative scripts (health probe, synchronous submit, async enqueue-and-poll) against a local server — **not run in CI**. No `config.py`/`main.py`/`pyproject.toml`/`app/api/` change (`httpx` is already a runtime dependency). Roadmap milestone M13 (client SDK).

- Analytics / feedback loop — platform stats seam + topic scorer (`backend/app/analytics/`) — the first component of the analytics/feedback layer (CLAUDE.md §3.4): pull published-video performance back in to steer what gets made next. Two deterministic halves kept distinct per §4. **(1) The seam** mirrors the search fabric point-for-point: a `@runtime_checkable` `AnalyticsProvider` Protocol (async `fetch_stats(*, post_id) -> VideoStats`), a strict `VideoStats` DTO, a hermetic `FakeAnalyticsProvider`, and a real httpx `YouTubeAnalyticsProvider`. `VideoStats` carries the platform's own `post_id` as the natural key (no synthetic id — the platform owns identity, unlike a discovered `Source`), `fetched_via`/`fetched_at` provenance (symmetric with `Source.discovered_via`; stats are tool-measured, never LLM-minted — §11), and keeps watch-time (`estimated_minutes_watched`, absolute) distinct from retention (`average_view_percentage`, a 0–100 ratio), both optional `| None` (None ≠ 0). - Scheduler / unattended batch runner (`backend/app/scheduler/`) — the automation seam CLAUDE.md §3.4 reserves for the "orchestration fabric": the loop that turns a backlog of topics into N videos/day with no human in the loop. Three deterministic, independently-testable *tools* (CLAUDE.md §4 — scheduling and batch execution are procedural, never agents) that compose into the loop. (1) `Schedule` (`schedule.py`) — **pure** next-run computation with no clock and no sleeping: `next_run_after(reference)` takes the reference instant *as an argument* (the caller owns the clock, mirroring `app.eval.harness`'s injected `time_fn`), so timing is unit-testable with zero real waiting. `IntervalSchedule(interval, anchor)` fires at **anchored** `anchor + k*interval` slots (stable/reproducible regardless of poll time); `DailySchedule(times)` fires at fixed **UTC** times of day (civil-tz/DST deferred). Both use **strictly-after** semantics (a reference exactly on a slot returns the *next* slot, so a fire-then-recompute loop never re-fires) and **reject naive datetimes** (the classic timezone bug, fail-loud per ADR 0001's tz-aware-UTC convention). (2) `TopicQueue` (`queue.py`) — the ordered backlog, FIFO by default and priority-aware, over `heapq` with a `(priority, seq, topic)` key whose monotonic `seq` tiebreaker is load-bearing (without it equal-priority entries fall through to comparing topic *strings*); lower priority produced first, ties FIFO; process-local/non-durable like `JobStore`. (3) `BatchRunner` (`runner.py`) — runs an **injected** `Produce = Callable[[str], Awaitable[T]]` (generic over `T`, decoupled from the real `VideoPipeline` a sibling builds — tests inject a fake; same workflow-agnostic seam as `JobStore`'s `JobRunner`) across a batch under an `asyncio.Semaphore` **created inside `run_batch`** (not `__init__`, so it binds to the running loop and tests can `asyncio.run` on a fresh loop), isolating per-topic failure into a captured `TopicResult.error` (one bad topic never aborts the batch, mirroring `JobStore`'s exception-to-terminal contract) and returning a `BatchResult` in **submission order**. **Deferred — the driver loop + real wiring:** the long-lived `while: sleep until next_run; drain; run_batch` process (the only piece that touches a real clock/sleep, and the site that binds `Produce` to the real `VideoPipeline` + publishing) is a documented follow-up — keeping it out keeps the three primitives pure and hermetic. Stdlib only (`asyncio`/`heapq`/`datetime` — apscheduler/celery deliberately avoided); no `config.py`/`main.py`/`pyproject.toml`/`api/` change. Fully hermetic tests (pure next-run math incl. strictly-after + naive-rejection boundaries; queue FIFO/priority; batch error isolation; concurrency cap forced deterministically with `asyncio.Barrier`, no sleeping). Decisions in [ADR 0034](docs/adrs/0034-scheduler.md). Automation / orchestration fabric (CLAUDE.md §3.4).

- Budget guardrails (`backend/app/services/budget/`) — estimated-spend metering + **enforcement** so unattended runs cannot rack up runaway API cost; the consumer ADR 0005 deferred "retries/budgets" to. A deterministic *service* (CLAUDE.md §4), not an agent — it meters and blocks; the Orchestrator owns *how* to react. Two composable pieces mirroring the fabric's decorator pattern (cache ADR 0026 / resilience ADR 0027): a `BudgetTracker` + pluggable `CostEstimator` accounting/enforcement core (`tracker.py`), and a `BudgetedModelProvider` `ModelProvider` decorator (`provider.py`). The tracker accrues call counts + estimated dollar cost scoped per-*run* and per-*calendar-day* (via an injected `Clock`); `charge(cost)` projects `accrued + cost` against each ceiling and raises `BudgetExceededError` **before** the cost is spent. Cost comes from a `CostEstimator` over a configurable `PriceTable`; two stdlib estimators ship — `PerCallEstimator` (flat $/call) and `TokenCostEstimator` ($/1k tokens via a pluggable `TokenCounter`, default the dependency-free `HeuristicTokenCounter`). The decorator estimates → reserves → blocks before the wrapped call, so a blocked call **never reaches the provider** and incurs no real spend (`name` wraps as `budgeted(...)`). Key decisions: **pre-call reservation** (accrue at check-time — closes the concurrency overshoot window, conservatively counts the attempt); **unmodeled model fails loud** (`UnknownModelError`, never silent $0 that would bypass the budget — opt into a catch-all via explicit `default_price`); **both ceilings optional + independent** (`per_run`/`per_day`, block if *either* breaks; the error carries `scope`/`limit`/`accrued`/`projected`); **calendar-day rollover** resets the daily tally, never the run tally; **strict `>`** boundary (projection *at* the ceiling is allowed). Estimate-vs-actual caveat documented: the provider contract returns no token-usage metadata, so enforcement is against a pre-call, input-only estimate — a conservative ceiling, not an invoice. Fully hermetic + deterministic: tests wrap a `FakeProvider` + a stub clock and assert the blocked call never reaches the provider. **Capability only — no wiring** (the M-LP pattern): no edits to `config.py`/`main.py`/`pyproject.toml`/`router.py`/`base.py`; adoption is a composition-root change owned by the Orchestrator consumer. Decisions in [ADR 0035](docs/adrs/0035-budget-guardrails.md).

A topic-submission form drives a typed `submitResearch` service (injectable `fetch` transport for mockability, snake-case wire contract mirroring the serialized `ResearchState`, ISO-string timestamps) and renders the result's four sections: plan sub-questions, discovered sources (with `discovered_via` provenance), synthesized findings, and the latest editorial critique. Presentation stays decoupled from the API (CLAUDE.md §10): small per-section presentational components receive resolved data and never call the service. The CLAUDE.md §11 evidence-vs-inference distinction is carried into the UI — a `SupportBadge` renders each finding's code-derived `weakest_support`, `disputed` findings are visually flagged, and the critique surfaces code-derived coverage gaps honestly. Ships an in-repo sample `ResearchState` fixture (and a "Load sample result" affordance) so the surface renders before the backend submit route lands in a sibling PR. No new dependencies; type-checks and builds via `npm run build`. Roadmap milestone M13 (frontend).
- Report generation (`backend/app/agents/report.py`, `backend/app/services/publishing/`, M11) — opens the Research Publishing band: composes the reasoning output (findings/verdicts/critique) into a structured, source-grounded `Report`. The agent/tool split (CLAUDE.md §4): the `ReportAgent` (`LONG_CONTEXT` role) authors prose (title, abstract, section narratives) referencing findings by local index, while deterministic `services/publishing/` tools own all structure — `citations` walks the provenance chain `Finding → Verdict → Evidence → Source` to assemble a snapshotted bibliography (the report is the *export* artifact, so it carries source-url snapshots per ADR 0001's attached-provenance rationale), and `caveats` derives the report's limitations. The §11 boundary is held at the final surface: **citations are code-derived** (a published report cannot cite a source the model invented), and **caveats are code-derived over the *full* findings set** (not just the cited subset — so the model cannot bury a contradiction by omitting it) plus the latest critique's uncovered sub-questions, quality issues, and an `UNRESOLVED_CRITIQUE` banner when the M10b revision loop exhausted unsatisfied (fulfilling ADR 0012's carry-forward promise). Adds `Report`/`ReportSection`/`Citation`/`Caveat`/`CaveatKind` schema and a `publishing` substate. Adds a dedicated `report` node (`plan → … → critique → report → publish`); the critique router's accept/exhausted arms now route to `report`, and `publish` becomes the lifecycle terminal (the success mirror of the failed sink). `ReportAgent` is the 8th `ResearchDeps` collaborator. A thin/all-disputed/exhausted report is valid (heavily caveated), not a failure; raises only on empty findings or zero resolvable sections. Markdown rendering + creator-packet fields deferred to M12. Decisions in [ADR 0017](docs/adrs/0017-report-generation.md). Roadmap milestone M11.
- Brave Search provider adapter (`backend/app/services/search/brave_search.py`, M-LP.2) — a second concrete `SearchProvider` for failover/robustness alongside the Tavily adapter (ADR 0013). An `httpx`-based adapter over the Brave Web Search API (`GET /res/v1/web/search`, `X-Subscription-Token` auth), mapping `web.results[]` → `SearchResult` (web type; Brave's `description` → snippet; URL-less hits skipped; `count` clamped to Brave's max of 20). The provider — never the LLM — mints the `url`, keeping the evidence-vs-inference boundary (CLAUDE.md §11; ADR 0006) structural. Operational failures (429/timeout/5xx via `raise_for_status`) propagate as `httpx` errors for the Orchestrator; only a malformed response *shape* is wrapped in a locally-defined `SearchError` (an absent/empty `web` block is a valid empty result, not an error). Adds `Settings.brave_api_key` (SecretStr), distinct from the LLM `api_key`. Fully offline-verifiable via `httpx.MockTransport`; live path covered by a `@pytest.mark.integration` smoke test. Per scope discipline only the new adapter module + config field are added — shared `base.py`/`__init__.py` are untouched, so the change stays merge-clean with the sibling Tavily branch. Decisions in [ADR 0021](docs/adrs/0021-brave-search-adapter.md). Roadmap milestone M-LP.2.
- Media Production Layer scaffold (`backend/app/media/`) — opens the second major component (CLAUDE.md §3.3) as provider-neutral, fully-faked *tool/service* seams (CLAUDE.md §4 — media work is deterministic execution, never agents; introduced via ADR per §3.4/§16). Three seams mirror the LLM/search fabric pattern: a `TTSProvider` protocol + hermetic `FakeTTSProvider` (text → `SynthesizedSpeech`), a `CompositionService` protocol + hermetic `FakeCompositionService` wrapping the future FFmpeg step (assets → `RenderedVideo`; **no real ffmpeg invoked**), and — deliberately asymmetric — a subtitle band that ships **real** code now: a synchronous `SubtitleService` protocol, a concrete `DeterministicSubtitleService`, and pure stdlib `format_srt`/`format_vtt` formatters (the band needs no external service). Typed artifact DTOs (`backend/app/media/schemas.py`: `SynthesizedSpeech`/`aud_`, `CaptionTrack`+`Caption`/`sub_`, `RenderedVideo`/`vid_`) are `extra='forbid'`, id-prefixed, and each top-level artifact carries a typed, required `produced_via` provenance string (symmetric with `Source.discovered_via`/`Evidence.extracted_via`; ADR 0006) — extending provenance integrity to the pipeline's tail. The layer imports nothing from `app.schemas`/`app.agents`/`app.workflows` (a documented local `_gen_id` copy keeps it decoupled), so it builds/tests/showcases standalone. ### Fixed

- Hardened three schema/test audit findings. **(1) Fixture drift** — the committed `research_state_minimal.json` fixture predated the `error`/`revision_iteration`/`reasoning`/`publishing` fields on `ResearchState` and the `test_json_shape_matches_committed_fixture` canary couldn't catch it (it only asserted the *parse* direction, so a newly-added field never broke the test). The fixture is regenerated to the full current shape and the canary now also asserts the **dump** direction (`json.loads(reference.model_dump_json()) == fixture`), so any future field added to the schema breaks the test and forces a deliberate fixture update. **(2) Unbounded eval score** — `QualityScore.score` (`app/eval/judge.py`) gained a `Field(ge=0)` lower bound (0..scale rubric; no upper bound, as `scale` is judge-configurable) so a negative judge score can no longer corrupt candidate ranking. **(3) Silent citation drop** — `assemble_citations` (`app/services/publishing/citations.py`) dropped a citation whose `source_id` did not resolve to a real `Source` *silently* (via a dead `source_type=None` accumulator that was filtered at the end, leaving an `Evidence`-snapshot fallback branch unreachable). It now skips the unresolved source before accumulating, **logs a warning** so the drop is observable, and the dead fallback/`assert` are removed (`source_type` is non-optional by construction). Focused regression tests added/strengthened for all three.
### Removed

- Dead Deep Research placeholder stubs (`backend/app/services/deep_research.py`, `backend/app/tools/deep_research.py`) — empty `pass`-body `DeepResearchService` / `DeepResearchTool` scaffolds superseded by the real `app/workflows/deep_research.py` graph + the agents. The `app/tools/` package keeps its `__init__.py` (not removed — it stays a valid package).

### Fixed

- Chunker redundant tail (`backend/app/services/ingestion/chunker.py`) — the fixed-window loop could emit a final chunk wholly contained in the previous chunk's overlap region (e.g. when the remainder ≤ overlap), a redundant duplicate tail. The loop now stops once a chunk reaches the end of the text (`start + window >= len(cleaned)`), since the would-be next window `[start+step, len) ⊂ [start, len)` adds no new content. Provably loses no characters; tests pin full per-character coverage of the input and a `window+1`-style input no longer producing a duplicate tail chunk.
### Fixed

- Event-loop-blocking parse in `IngestionService.ingest` (`backend/app/services/ingestion/service.py`) — the `async def ingest` called the synchronous, CPU-bound `parse_html(...)` and `PdfParser.parse(...)` directly on the event loop, so a large HTML/PDF parse blocked the entire loop and stalled every other research job, status poll, and `/health` while it ran. Both blocking calls now run off the loop via `await asyncio.to_thread(...)`, mirroring the established pattern in `app/media/composition/ffmpeg.py` and the YouTube transcript provider. Public behavior is unchanged: the worker-thread exception re-raises at the `await`, so the same per-source skip-and-log `except (FetchError, ParseError, TranscriptError)` handling (and the no-chunks `IngestionError`) is preserved; the existing ingestion tests cover the path.
