---
name: adr-writer
description: Use this skill when an implementation introduces or changes a meaningful architectural decision. Do not use for tiny local edits.
---

Write an ADR in docs/adrs using this structure:
- Title
- Date
- Status
- Context
- Decision
- Alternatives considered
- Consequences
- Follow-up actions

Guidelines:
- Keep it concrete.
- Reference actual modules/files involved.
- Explain why rejected alternatives were not chosen.
- Prefer one ADR per decision.
